Foreword

The software developed by SmartCampus is being made available as Open Source. This page provides some documentation about the approach that has been followed and about how to take full advantage of the public release of SmartCampus code base. The components of the overall SmartCampus code base are published under the Apache License Version 2.0 (​http://www.apache.org/licenses/LICENSE-2.0.html). The access to the source code is free.

Open Source approach

The open source release of the project code base is one of the way by which SmartCampus is trying to build an pro-active community around the design and development of services for the students of the University of Trento.

The goal of the SmartCampus open source release is to allow anybody within our growing community to actively contribute to the development of new services. More precisely, the goal of this initial release is to enable the development of Android client applications based on the SmartCampus platform. Future releases of the code base will grant the possibility to contribute also to the development of the SmartCampus platform itself.

Open Source structure

The infrastructure of the Open Source release consists of:
  • a set of SVN repositories
  • a Nexus artifact repository
  • a TRAC instance

Currently, the release involves four different kinds of artifacts:

  • VAS (Value Added Service): the services implemented within the SmartCampus project (splitted into components):
    • Android client
    • web application
    • domain libraries
    • libraries for wrapped external services
  • platform components in two flavours:
    • as source code
    • as libraries (on a artifact repository)
  • common libraries for Android client apps
  • wrapping of external services

SVN

The code is stored into a set of ​Subversion (SVN) repositories.

The SVN repositories can be browsed on the web at ​http://svn.smartcampuslab.it/svn/

The contents of the SVN repositories is available for download at the following URLs:

  • platform:
    • the components of the SmartCampus platform that are needed to execute services
    • http://svn.smartcampuslab.it/svn/platform
  • clientlibs:
    • the Android common libraries that are needed to execute the services
    • http://svn.smartcampuslab.it/svn/clientlibs
  • services:
    • the external services wrapped into the platform to be used by the VAS
    • http://svn.smartcampuslab.it/svn/services
  • VAS:
    • the services developed by SmartCampus
    • communicator: ​http://svn.smartcampuslab.it/svn/vas/communicator
    • communitymanager: ​http://svn.smartcampuslab.it/svn/vas/communitymanager
    • journeyplanner: ​http://svn.smartcampuslab.it/svn/vas/journeyplanner
    • discovertrento: ​http://svn.smartcampuslab.it/svn/vas/discovertrento
    • experiencebuster: ​http://svn.smartcampuslab.it/svn/vas/experiencebuster
    • portfolio: ​http://svn.smartcampuslab.it/svn/vas/portfolio
    • launcher: ​http://svn.smartcampuslab.it/svn/vas/launcher
To check out the code from any of these repositories(anonymous, read-only Git access), do this: git clone For instance to checkout a platform project the command to be issued is: git clone https://github.com/smartcampuslab/clientlibs-AndroidJourneyPlannerLib.git To do the clone, you’ll need Git.

Nexus

Some portions of the code that are used by the platform/services, but that are not part of SmartCampus production, are made available, under their own license, as libraries on a Nexus instance. The downloadable packages, depending on the artifact, may include one or more of these subpackages:

  • source code snapshot
  • binaries
  • API docs

TRAC

If you are here you are probably already aware of the existence of a SmartCampus TRAC system linked to the SmartCampus SVN repositories. This instance of TRAC is common to all the repositories and is used to provide:
  • a wiki where to supply documentation about the project source code
  • a ticketing system where to track bugs and enhancements requests
